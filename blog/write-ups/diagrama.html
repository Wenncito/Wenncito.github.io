Guía de Bug Bounty para Páginas Web

  • Reconocimiento
    • Recolección de Información Pública
      • Utiliza herramientas como Whois, Google Dorks, y redes sociales.
    • Mapeo de la Superficie de Ataque
      • Identifica todos los puntos de entrada posibles (subdominios, IPs, etc.).
    • Enumeración de Subdominios
      • Usa herramientas como Sublist3r, Amass, etc.
  • Escaneo de la Página Web
    • Escaneo de Puertos
      • Utiliza nmap para identificar puertos abiertos.
    • Detección de Servicios
      • Determina qué servicios están corriendo en cada puerto.
    • Análisis de Vulnerabilidades
      • Usa herramientas como Nessus o OpenVAS.
  • Identificación de Tecnologías
    • Identifica las tecnologías usadas en la página web (CMS, frameworks, librerías, etc.) usando Wappalyzer, BuiltWith, etc.
  • Vulnerabilidades Comunes
    • Familiarízate con OWASP Top 10 y CWE/SANS Top 25.
      • Explora vulnerabilidades específicas relacionadas con las tecnologías identificadas.
  • Exploración de Puntos de Entrada
    • Busca formularios, URLs paramétricas, y cualquier punto de entrada de datos del usuario.
  • Pruebas de Inyección
    • Realiza pruebas de inyección SQL, XSS, y otras técnicas de inyección.
  • Pruebas de Autenticación y Autorización
    • Verifica si puedes eludir mecanismos de autenticación y si hay problemas de control de acceso.
  • Pruebas de Configuración
    • Busca configuraciones erróneas en servidores, archivos sensibles expuestos, etc.
  • Pruebas de Lógica de Negocio
    • Analiza la lógica de la aplicación para encontrar errores que podrían ser explotados.
  • Análisis de Resultados
    • Revisa y documenta los hallazgos de tus pruebas.
  • Reporte de Vulnerabilidades
    • Prepara un informe detallado con tus descubrimientos y envíalo al programa de Bug Bounty.
